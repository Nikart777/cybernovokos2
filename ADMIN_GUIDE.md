# Инструкция по управлению ценами (Admin Panel)

Для проекта CyberX Novokosino была разработана простая система управления контентом (CMS) без использования баз данных. Все данные хранятся в файле `data/prices.json`.

## 1. Подготовка
Перед использованием убедитесь, что в файле `.env.local` или в настройках деплоя на Rusonix установлена переменная:
`ADMIN_PASSWORD=ваш_секретный_пароль`

## 2. Как изменить цены
1. Перейдите по адресу: `https://cyberx-novokosino.ru/admin/prices` (или `localhost:3000/admin/prices` при локальной разработке).
2. Введите пароль администратора.
3. В интерфейсе админки:
   - **Редактируйте**: можно менять названия зон, описания железа, время и стоимость (будни/выходные).
   - **Добавляйте**: используйте кнопки "+" для создания новых тарифов, категорий или зон.
   - **Удаляйте**: иконка корзины удаляет соответствующий элемент.
   - **Ночь**: кнопка "НОЧЬ" помечает тариф как ночной (добавляет специальный дизайн на страницу).
4. Нажмите кнопку **"Сохранить"** в верхней части экрана.

## 3. Применение изменений
Поскольку проект использует статическую генерацию или серверный рендеринг:
- **Локально**: изменения в `data/prices.json` сохраняются мгновенно. Вам нужно сделать `git commit` и `git push`, чтобы изменения попали в репозиторий.
- **В продакшене**: после сохранения в админке данные запишутся в файл на сервере. Однако для того, чтобы изменения стали постоянными в коде, рекомендуется закоммитить обновленный `data/prices.json` из репозитория после локальных правок.

> [!TIP]
> Если вы меняете цены прямо на живом сайте через админку, они обновятся сразу (так как страница `/prices` читает JSON при каждом запросе). Но при следующем деплое из GitHub данные могут затереться старой версией из репозитория, поэтому **всегда дублируйте важные изменения в коде**.

## 4. Безопасность
- Путь `/admin/*` закрыт от индексации поисковиками в `robots.ts`.
- Все API запросы на изменение данных требуют проверки пароля на стороне сервера.
